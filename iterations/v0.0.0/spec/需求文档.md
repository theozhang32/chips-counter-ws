# 筹码计数器 — 产品需求文档

> 版本：v0.0.0（草案，待审查）  
> 说明：本文仅描述产品目标、使用场景与功能需求，不涉及任何技术实现、架构或选型。

---

## 1. 背景与问题

线下进行德州扑克等筹码类桌游时，玩家常因未携带实体筹码而无法正常开局，或只能用临时替代物（现金、纸笔、手机备忘录）记录，过程混乱、易出错、难以对账。

本应用旨在用数字化方式替代实体筹码的「数量记录」职能，让玩家在没有实体筹码时也能顺利开局与结算。

---

## 2. 产品目标

帮助一局游戏中的多名玩家，共同维护一份可信、可对账的筹码记录，覆盖开局买入、局中增减、局末结算的完整过程。

**成功标准（产品层面）：**

- 没有实体筹码时，玩家仍可完成一局完整的筹码记录与结算
- 每位玩家随时能看到自己与他人的当前筹码量
- 局末可清晰核对各自盈亏，减少口角与扯皮

---

## 3. 产品边界

### 3.1 做（In Scope）

- 记录玩家身份与昵称
- 记录每位玩家的筹码数量变化
- 支持开局买入、局中补充/提取、局末结算与盈亏展示
- 支持一局多人共同查看与更新同一份筹码账本

### 3.2 不做（Out of Scope）

本应用**不参与任何游戏规则判定**，包括但不限于：

- 发牌、大小盲、下注轮次、跟注/加注/弃牌等流程
- 牌型比较、胜负判定、边池计算
- 强制按牌局规则自动扣减或转移筹码
- 计时器、座位轮转、庄家位管理等牌局流程辅助

上述事项由玩家线下自行商定与执行；应用只忠实记录玩家主动确认的筹码变动结果。

### 3.3 非目标（当前版本不考虑）

- 真实货币支付、充值、提现
- 匹配陌生人开局、公开展示对局
- 牌谱记录、复盘分析
- 锦标赛结构、盲注表、奖励分配算法

---

## 4. 用户与角色

| 角色 | 说明 |
|------|------|
| 房主 / 发起人 | 创建一局游戏，邀请他人加入，可结束本局 |
| 玩家 | 加入一局后，查看全员筹码，发起或确认与自己相关的筹码变动 |
| 观察者（可选） | 仅查看筹码情况，不参与变动（若首版不做，可标注为后续） |

首版可默认：所有加入者均为玩家；房主同时具备玩家身份。

---

## 5. 核心使用场景

### 场景 A：开局组局

1. 一名玩家发起一局游戏
2. 设定本局基础信息（如局名、初始买入建议额等，具体字段见功能需求）
3. 其他玩家加入该局
4. 各玩家确认自己的初始买入筹码数量
5. 全员就绪后，本局进入进行中状态，筹码账本生效

### 场景 B：局中筹码变动

常见变动类型（均由玩家主动操作，应用不做规则校验）：

- **补充买入（Rebuy）**：玩家增加筹码
- **提取筹码（Cash-out 部分）**：玩家减少当前桌上筹码（例如提前离席一部分）
- **玩家间转移（可选）**：将一定数量从 A 转给 B（用于线下已完成的输赢兑现记录）

每次变动应留下可追溯记录，便于事后核对。

### 场景 C：局末结算

1. 房主或约定角色发起结算
2. 确认各玩家最终筹码数量（可与系统当前记录核对）
3. 按「最终筹码 − 累计买入」计算各人盈亏并展示
4. 本局归档，之后仅可查看、不可再改（或仅允许更正说明，见待决事项）

### 场景 D：中途加入 / 离开

- 中途加入：新玩家完成初始买入后进入账本
- 中途离开：可选择结算该玩家当前盈亏后离席，或保留其筹码记录至局末（行为需在审查中确认）

## 7. 业务规则（产品规则，非牌局规则）

1. **筹码仅为记账单位**：应用内数字不代表法定货币，也不强制与现金一一对应；兑换比例由玩家线下约定。
2. **变动主动发生**：任何筹码数量变化，必须由有权限的用户主动发起并确认，系统不因「赢了一手」等牌局事件自动改数。
3. **累计买入与当前筹码分离**：
   - 累计买入：本局该玩家投入桌上的筹码总量（含初始与 rebuy）
   - 当前筹码：该玩家此刻桌上持有的筹码量
   - 盈亏 = 当前（或结算时最终）筹码 − 累计买入
4. **零和原则**：在无外部注入/抽出记账错误的前提下，全员盈亏之和应为 0；应用应能发现并提示不平账。
5. **一局生命周期**：未开始 → 进行中 → 已结算；已结算后默认不可再改动筹码（更正流程见待决事项）。

## 8. 非功能期望（产品层面）

| 类别 | 期望 |
|------|------|
| 可靠性 | 对局进行中数据不应无故丢失；关键操作结果应对参与者一致可见 |
| 易用性 | 不懂技术的玩家也能在一分钟内完成「加入并买入」 |
| 透明度 | 每人都能看到同一套数字与流水，减少信息不对称 |
| 隐私 | 对局内容默认仅对参与者可见，不对外公开 |

（不在此规定具体性能指标、平台或实现方式。）

## 9. 待决事项（需审查确认）

请在审查时重点拍板以下问题：

1. **身份方式**：是否需要登录/账号？还是「创建局 + 口令/链接」即可匿名参与？
2. **改数权限**：谁可以修改谁的筹码？是否必须对方确认？房主是否有强改权限？
3. **玩家间直接转账**：首版是否支持「A 转给 B」？还是只支持各自买入/提取，输赢由线下口头完成后各自改自己的数？
4. **中途离席**：离席时是否强制结算该玩家？离席后是否仍显示在名单中？
5. **观察者角色**：首版是否需要只读旁观？
6. **已结算更正**：发现记错后，是否允许房主开启「更正」并留痕？
7. **多局/常客**：是否需要记住常一起玩的好友，还是每局临时组局即可？
8. **筹码单位与精度**：仅整数？是否允许小数？是否需要自定义单位名称（如「分」「BB」）？

---

## 10. 验收标准（首版建议）

完成以下闭环即可视为首版达标：

1. 多人可进入同一局并各自完成初始买入  
2. 局中可补充买入，全员看到更新后的余额  
3. 可查看变动流水  
4. 可结束对局并看到每人盈亏，且零和校验生效  
5. 全程无任何牌局胜负、牌型或下注流程相关能力
